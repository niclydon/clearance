# Contributing

Clearance is documentation-first because it is public infrastructure.

The project should be usable by someone who does not know the Nexus origin story and does not share the maintainer's local operating context.

## Start Here

- [Documentation standards](documentation-standards.md)
- [Implementation workflow](implementation-workflow.md)

## Contribution Principles

- Keep the PMO model portable.
- Document behavior as it lands.
- Preserve the difference between lifecycle status and execution permission.
- Do not make chat platforms, LLM providers, or local deploy systems required.
- Keep examples concrete and operational.
- Mark planned behavior until it is implemented and tested.

## Current Status

Contribution docs are intentionally early. They should become more specific as package structure, tests, and release workflows land.
