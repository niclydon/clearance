# First Project

A project track is the durable home for a major workstream.

## Example

Imagine a homelab operator wants to improve backup reliability.

The first project track might be:

- key: `homelab-backup-reliability`
- title: `Homelab backup reliability`
- summary: `Track backup health checks, restore tests, alerting gaps, and documentation.`
- status: `active`

The track can then link to work items, docs, deploy runs, incidents, and candidate discoveries.

## What Good Looks Like

A good first project track has:

- a stable name
- a clear reason to exist
- a first next action
- links to evidence as work begins
- enough scope to group related work without becoming a vague forever bucket
